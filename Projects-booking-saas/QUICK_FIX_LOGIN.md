# Быстрое исправление логина

## Проблема: 500 Internal Server Error при логине

### Возможные причины:

1. **JWT_SECRET не установлен в .env**
2. **База данных не подключена**
3. **Таблица owners не существует**

## Решение:

### 1. Проверь файл `.env` в корне проекта

Должны быть установлены:
```env
DATABASE_URL=mysql://user:password@localhost:3306/db_name
JWT_SECRET=любая_случайная_строка_минимум_32_символа
```

### 2. Если JWT_SECRET отсутствует, добавь:

```env
JWT_SECRET=your-super-secret-jwt-key-min-32-chars-long-123456789
```

### 3. Проверь, что таблица owners существует:

Запусти миграцию:
```bash
pnpm db:push
```

### 4. Перезапусти сервер:

После изменений в .env перезапусти `pnpm dev`

## После исправления:

Попробуй войти с любым email - система автоматически создаст аккаунт.
